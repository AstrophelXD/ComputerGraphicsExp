# 实验4：多面体的观察投影及动画显示

根据 `docs/图形实验四.docx` 的要求，本目录实现了最小 MFC 版本的多面体观察投影与动画显示。

## 实现内容

- 仅实现 21 组及以后要求：十二面体（Dodecahedron）线框显示；
- 使用顶点表 + 面表（`CFace`）定义多面体结构；
- 在 `PerProject()` 中完成观察变换与透视投影；
- 在 `OnTimer()` 中持续更新旋转角，实现动画显示。

## 编译运行

在仓库根目录执行：

```bat
build_mfc.bat exp4
```

构建成功后运行 `exp4/exp4.exe`。

## 已知限制

- 当前版本按题目要求提供线框动画，不包含消隐与光照；
- 为兼容当前统一构建脚本，`main.cpp` 内包含了 `Face.cpp`。

## 附录：`main.cpp` 原理详解（按模块）

本附录按“数据建模 -> 投影算法 -> 动画驱动 -> MFC 生命周期”的顺序，逐模块说明 `exp4/main.cpp` 如何实现十二面体观察投影动画。

### 1) 依赖与编译组织模块

- `afxwin.h`：提供 MFC 的应用、窗口、视图、消息映射和 `CDC` 绘图接口；
- `<array>`：存储固定大小的顶点数组；
- `<cmath>`：提供 `sin/cos/sqrt` 等数学计算；
- `Face.h` + `P3.h`：分别提供面结构 `CFace` 与三维点结构 `CP3`；
- `#include "Face.cpp"`：由于当前 `build_mfc.bat` 仅编译 `main.cpp`，这里把 `CFace` 实现并入当前编译单元，避免链接缺失。

该模块目标是“最小可编译交付”：不改统一脚本前提下确保 `exp4` 独立构建通过。

### 2) 视图状态与核心数据模块

`CExp4View` 中核心状态包括：

- `m_vertices`：20 个十二面体顶点（三维模型坐标）；
- `m_faces[12]`：12 个五边形面，每面存储顶点索引；
- `m_r / m_d`：观察投影参数（视点距离与投影距离）；
- `m_thetaDeg / m_phiDeg / m_tk[9]`：观察变换角度与 3x3 变换系数；
- `m_rotY / m_rotX`：模型动画旋转角（Y 轴主旋转 + X 轴辅旋转）。

设计上把“几何数据”“投影参数”“动画状态”解耦，便于后续单独调节视角或动画速度。

### 3) 多面体建模模块：`InitDodecahedron()`

该函数完成十二面体拓扑初始化：

1. 根据黄金比例 `phi=(1+sqrt(5))/2` 构造标准十二面体顶点集合；
2. 按比例系数 `s` 放大到合适显示尺寸；
3. 通过 `faceIndex[12][5]` 给出 12 个五边面；
4. 对每个 `CFace` 调用 `SetNum(5)` 并写入索引。

这正对应题目“定义顶点表和面表”的要求，且仅保留 21 组及以后对应形体（十二面体）。

### 4) 观察变换参数模块：`InitParameter()`

该模块根据 `theta/phi` 计算观察坐标系系数 `m_tk[0..8]`：

- 先角度转弧度；
- 预计算 `sin/cos`；
- 按课程观察变换公式填充 3x3 旋转映射。

输出结果供后续所有点投影复用，避免每个点重复推导公式。

### 5) 模型动画旋转模块：`RotateModel(const CP3&)`

动画旋转在投影前执行，流程为：

1. 先绕 Y 轴旋转（主运动）；
2. 再绕 X 轴旋转（增加空间层次）；
3. 返回旋转后的三维点。

这样可用较少参数获得稳定、可观察的三维动态效果，满足“动画显示”要求。

### 6) 透视投影模块：`PerProject(const CP3&)`

该函数实现“模型点 -> 视图点 -> 屏幕点”的完整映射：

1. 先调用 `RotateModel` 得到当前帧模型点；
2. 用 `m_tk` 变换到观察坐标系，得到 `xView/yView/zView`；
3. 做深度平移 `zView + m_r`；
4. 透视除法得到二维坐标：
   - `x = m_d * xView / zView`
   - `y = m_d * yView / zView`
5. 对 `zView` 近零情况做保护，避免除零异常。

该模块是实验四“观察投影”的核心数学实现。

### 7) 辅助坐标轴模块：`DrawAxis(CDC*)`

- 先投影原点与 X/Y/Z 轴末端点；
- 再用 `MoveTo/LineTo` 绘制三条坐标轴；
- `TextOut` 标注轴名。

坐标轴用于直观验证视角、投影方向和动画旋转是否符合预期。

### 8) 面表线框绘制模块：`DrawObject(CDC*)`

该模块按“面 -> 边”遍历绘制线框：

1. 遍历 12 个面；
2. 对每个面按相邻顶点索引连边；
3. 对每条边的两个端点分别做 `PerProject`；
4. 调用 GDI 线段绘制。

实现上使用面表直接出边，简单直观，符合题目“显示每个面的每条边”描述。

### 9) MFC 生命周期与消息模块

#### `OnCreate(...)`

- 初始化十二面体数据 `InitDodecahedron()`；
- 启动定时器 `SetTimer(1, 30, nullptr)`，约 33 FPS。

#### `OnTimer(...)`

- 周期累加 `m_rotY/m_rotX`；
- 对角度做 `2*pi` 回绕；
- `Invalidate(FALSE)` 请求重绘，形成连续动画。

#### `OnDestroy(...)`

- 关闭定时器 `KillTimer(1)`，完成资源清理。

这构成典型的“定时驱动 -> 更新状态 -> 触发重绘”动画循环。

### 10) 主绘制入口模块：`OnDraw(CDC*)`

每帧绘制顺序如下：

1. 获取客户区并白底清屏；
2. 设置 `MM_ANISOTROPIC`；
3. 通过 `SetViewportExt(width,-height)` 令 Y 轴向上；
4. 通过 `SetViewportOrg(center)` 把原点移到窗口中心；
5. 调用 `InitParameter()` 更新当前观察系数；
6. 绘制坐标轴与十二面体线框；
7. 输出标题文字用于实验标识。

因此最终画面是“以窗口中心为原点、Y 向上、带透视缩小”的十二面体线框动画。

### 11) 应用外壳模块（`Frame/App`）

- `CExp4Frame`：创建主窗口并设置标题；
- `CExp4App::InitInstance()`：
  - 初始化 MFC；
  - 创建 `Frame` 与 `View`；
  - 挂接视图为主窗口内容区；
  - 显示并刷新窗口。

该层保持最小化，仅负责窗口生命周期管理，不耦合图形算法。

### 12) 模块协作总流程

运行时完整链路如下：

1. `InitInstance` 创建主窗口与视图；
2. `OnCreate` 建立十二面体数据并启动定时器；
3. `OnTimer` 周期更新旋转角；
4. `OnDraw` 依据当前角度进行投影并绘制线框；
5. 周期循环，形成稳定动画显示。

对应实验目标即：多面体观察投影（线框） + 动画显示。
