# 实验2：多面体观察投影

根据 `docs/图形实验二.docx` 的要求，本目录实现了一个最小 MFC 版本的多面体观察投影线框显示。

## 实现内容

- 在 `CView::OnDraw` 中设置映射模式并绘制坐标轴；
- 实现观察变换参数计算 `InitParameter()`；
- 实现透视投影函数 `PerProject()`；
- 实现线框绘制函数 `DrawObject()`；
- 仅实现 21 组及以后要求：八面体线框观察投影（固定实现，不提供组号输入）。

## 编译运行

推荐在 Visual Studio 安装 MFC 组件后，使用开发者命令行编译：

```bat
cd /d d:\GraphicsExp\exp2
cl /nologo /std:c++17 /EHsc /MD /D_AFXDLL /Fe:exp2.exe main.cpp /link /NOLOGO /SUBSYSTEM:WINDOWS
```

运行 `exp2.exe` 后即可看到坐标轴与八面体的观察投影视图。

## 已知限制

- 当前版本仅显示线框，不含消隐与光照；
- 仅保留 21 组及以后对应内容，不包含四面体/六面体分支。

## 附录：`main.cpp` 原理详解（按模块）

本附录按代码模块说明程序如何把三维八面体转换为二维线框图，并在 MFC 窗口中显示。

### 1) 头文件与基础类型模块

- `#include <afxwin.h>`：提供 MFC 应用、窗口、视图和设备上下文（`CDC`）相关能力；
- `#include <array>`：用于定长顶点/边数组；
- `#include <cmath>`：用于三角函数和绝对值计算；
- `struct CP3`：三维点结构（`x/y/z`），用于表示模型点、视图点、投影点；
- `struct Edge`：边结构（`s/e`），存储一条边在顶点数组中的起止索引。

设计动机：把“几何数据”与“绘制流程”解耦，便于后续更换模型而不改投影算法。

### 2) 常量与视图状态模块

- `kPi`：角度转弧度时使用的常量；
- `CExp2View` 成员：
  - `m_r`：视点到观察坐标系原点的距离（平移到视点坐标）；
  - `m_d`：投影距离（透视强度）；
  - `m_thetaDeg/m_phiDeg`：视点方向角（方位/俯仰）；
  - `m_tk[9]`：观察变换系数（3x3 旋转部分按一维展开存储）。

设计动机：把观察参数做成成员变量，统一由 `InitParameter()` 计算，避免在每次投影时重复推导。

### 3) 几何数据定义模块（八面体）

- `OctaVertices()`：返回 6 个顶点（上下两个尖点 + 赤道四点）；
- `OctaEdges()`：返回 12 条边，完整连接形成八面体线框。

当前实现只保留八面体，是因为实验要求限定为 21 组及以后，且不需要组号输入。

### 4) 观察变换参数计算模块：`InitParameter()`

作用：根据 `theta/phi` 计算观察坐标系基向量，生成 `m_tk` 系数。

步骤：

- 把角度转弧度：`theta = m_thetaDeg * pi / 180`，`phi = m_phiDeg * pi / 180`；
- 预计算 `sin/cos(theta)` 与 `sin/cos(phi)`；
- 按课程公式填写 `m_tk[0..8]`，构成从世界坐标到观察坐标的旋转映射。

结果：后续任意三维点都可通过 `m_tk` 快速转换到观察坐标系。

### 5) 透视投影模块：`PerProject(const CP3&)`

作用：把三维点映射到二维屏幕平面。

计算流程：

1. 观察变换：先得到 `xView/yView/zView`；
2. 深度平移：`zView += m_r`，避免对象落在投影中心附近导致不稳定；
3. 透视除法：
   - `xScreen = m_d * xView / zView`
   - `yScreen = m_d * yView / zView`
4. 当 `zView` 过小（接近 0）时进行保护，避免除零。

直观上，`zView` 越大，投影点越靠近中心，体现“近大远小”。

### 6) 坐标轴绘制模块：`DrawAxis(CDC*)`

作用：绘制三维坐标系在当前视角下的投影，便于观察空间方向。

流程：

- 先将原点和三条轴末端点分别送入 `PerProject()`；
- 使用 `MoveTo/LineTo` 画三条轴线；
- 使用 `TextOut` 标注 `X/Y/Z`。

该模块是调试观察参数是否合理的直观参照。

### 7) 通用线框绘制模块：`DrawWire(...)`

作用：将“顶点数组 + 边数组”渲染为二维线段集合。

流程：

- 遍历每条 `Edge`；
- 用边的 `s/e` 索引取对应起点/终点三维坐标；
- 分别投影到二维；
- 用 `MoveTo/LineTo` 连线。

优点：绘制逻辑和具体形体解耦，若以后替换为其他线框体，只需换数据数组。

### 8) 对象绘制入口模块：`DrawObject(CDC*)`

当前行为：固定调用 `DrawWire(pDC, OctaVertices(), OctaEdges())`，只绘制八面体。

这正对应实验二在本仓库的交付范围（21+）。

### 9) 主绘制流程模块：`OnDraw(CDC*)`

`OnDraw` 是本实验图形结果的总入口，流程如下：

1. 获取客户区并清屏为白色；
2. 设置 MFC 映射模式：
   - `MM_ANISOTROPIC` 允许自定义窗口/视口比例；
   - `SetViewportExt(..., -height)` 把 Y 轴改为“向上为正”；
   - `SetViewportOrg(center)` 把原点移动到窗口中心；
3. 执行 `InitParameter()`；
4. 绘制坐标轴 `DrawAxis()`；
5. 绘制八面体线框 `DrawObject()`。

因此，最终画面是“以窗口中心为原点、Y 轴向上”的二维投影视图。

### 10) MFC 框架模块（应用/窗口/视图）

- `CExp2View`：承载核心绘制算法与投影逻辑；
- `CExp2Frame`：主窗口类，负责窗口创建与标题；
- `CExp2App::InitInstance()`：
  - 初始化 MFC 应用；
  - 创建 `Frame` 和 `View`；
  - 绑定视图为主窗口内容区；
  - 显示并更新窗口。

这一结构对应最小 MFC 单文档风格，满足“代码精简、功能完整”的实验实现目标。
