# 实验3：样条曲线动画图形显示

依据 `docs/图形实验三.docx`，本目录仅实现 **21组及以后** 需要完成的内容：

- 不实现 1-15 组的 Bezier 分支；
- 仅实现 B 样条曲线图形显示；
- 对控制点做二维变换并循环重绘，形成动画（含延时与清屏）。

## 实现说明

- 文件：`main.cpp`（MFC 最小结构：`CWinApp + CFrameWnd + CView`）；
- 曲线：三次均匀 B 样条，按四点一段进行数值采样绘制；
- 图形：以叶片轮廓控制点为示例，控制多边形和曲线同时显示；
- 动画：定时器周期触发，对控制点执行旋转 + 非等比缩放 + 平移，再重绘。

## 编译运行

在仓库根目录执行：

```bat
build_mfc.bat exp3
```

生成 `exp3/exp3.exe` 后运行即可。

## 当前范围与限制

- 仅保留 21 组及以后对应要求（B 样条及动画）；
- 控制点当前为内置示例点，未接入 `B_B` 工具导出的文件输入。

## 附录二：`main.cpp` 原理详解（按模块）

本附录按“数据 -> 几何算法 -> 动画驱动 -> MFC 生命周期”的顺序，逐模块解释 `exp3/main.cpp` 如何完成 B 样条动画显示。

### 1) 基础依赖与数据结构模块

- `afxwin.h`：提供 MFC 应用、窗口、视图、消息映射与 `CDC` 绘图接口；
- `<vector>`：存储控制点集合与动画后点集；
- `<array>`：当前文件保留通用容器依赖（若后续扩展固定控制点组可直接复用）；
- `<cmath>`：提供 `sin/cos/fmod` 等动画与变换计算；
- `Point2`：二维点结构（`x/y`），统一用于控制点、变换点、曲线采样点；
- `kPi`：周期动画与角度换算常量。

设计目的：先把“数据表示”独立出来，让后续所有几何运算都围绕统一点结构进行，避免重复转换。

### 2) 视图状态与控制点集合模块

`CExp3View` 中核心状态有三类：

- `m_baseCtrl`：静态基准控制点（图形原始轮廓）；
- `m_animCtrl`：每帧变换后的控制点（用于实际绘制）；
- `m_phase`：动画相位（时间参数）。

运行逻辑是“**基准点不动，逐帧计算动画点**”。这样可以避免误差累计（若直接反复改写同一组点会产生漂移）。

### 3) 几何辅助模块：中心点与二维仿射变换

#### `ComputeCenter(...)`

- 对点集求平均，得到图形几何中心；
- 后续旋转与缩放均围绕该中心执行，防止图形绕窗口原点公转。

#### `TransformPoint(...)`

该函数将二维变换拆成清晰步骤：

1. 平移到中心局部坐标：`(x - cx, y - cy)`；
2. 先做非等比缩放：`sx/sy`；
3. 再做旋转（`cos/sin`）；
4. 平移回原位置并叠加位移 `tx/ty`。

这样实现比“直接拼矩阵”更直观，且便于单独调节每个动画分量。

### 4) 动画控制点更新模块：`UpdateAnimatedControlPoints()`

此模块是动画行为的核心：

- 用 `m_phase` 生成周期参数；
- 构造旋转角 `angle`、缩放 `sx/sy`、平移 `tx/ty`；
- 遍历 `m_baseCtrl`，调用 `TransformPoint` 生成 `m_animCtrl`。

当前参数设计为不同频率的正弦组合，目的是让动画看起来更自然，避免纯匀速旋转的机械感。

### 5) 曲线数学模块：`EvalUniformCubicBSpline(...)`

该函数实现三次均匀 B 样条单段求值，输入 4 个控制点和参数 `u in [0,1]`：

- 先算 `u^2`、`u^3`；
- 计算 4 个基函数系数 `b0~b3`；
- 使用加权和得到曲线点坐标：
  - `x = b0*p0.x + b1*p1.x + b2*p2.x + b3*p3.x`
  - `y = b0*p0.y + b1*p1.y + b2*p2.y + b3*p3.y`

这是课程“根据方程画曲线”的直接实现，不依赖库级样条封装。

### 6) 控制多边形绘制模块：`DrawControlPolygon(...)`

- 作用：显示控制点连线，便于观察曲线与控制点关系；
- 采用灰色虚线笔，视觉上与主曲线区分；
- 逐段 `MoveTo/LineTo` 绘制折线。

该模块不是必须交付项，但对实验演示和调试非常有价值。

### 7) B 样条绘制模块：`DrawBSpline(...)`

流程如下：

1. 遍历控制点窗口 `[i, i+1, i+2, i+3]`，每 4 点形成一段样条；
2. 每段用 `stepsPerSeg` 做均匀采样；
3. 对每个 `u` 调 `EvalUniformCubicBSpline` 得到采样点；
4. 将采样点顺次连线形成平滑曲线。

参数 `stepsPerSeg` 越大曲线越平滑，但绘制开销也越高；当前值在平滑度与性能间取平衡。

### 8) MFC 生命周期与消息模块

#### `OnCreate(...)`

- 初始化 `m_animCtrl`；
- 启动定时器 `SetTimer(1, 40, nullptr)`（约 25 FPS）。

#### `OnTimer(...)`

- 更新 `m_phase`；
- 调 `UpdateAnimatedControlPoints()` 生成新一帧控制点；
- `Invalidate(FALSE)` 请求重绘（不擦除背景，降低闪烁）。

#### `OnDestroy(...)`

- 调 `KillTimer(1)` 清理定时器资源。

这套流程形成“定时驱动 -> 更新状态 -> 重绘显示”的经典动画闭环。

### 9) 主绘制入口模块：`OnDraw(CDC*)`

`OnDraw` 是每帧最终输出入口，步骤为：

1. 取客户区并白底清屏；
2. 设置 `MM_ANISOTROPIC`；
3. 使用 `SetViewportExt(width, -height)` 将 Y 轴方向改为“向上为正”；
4. `SetViewportOrg(center)` 将坐标原点放在窗口中心；
5. 绘制控制多边形；
6. 绘制 B 样条曲线；
7. 输出简单标题文字。

这样可使实验坐标系与数学平面直觉一致，便于按控制点坐标直接调图。

### 10) 应用外壳模块（`Frame/App`）

- `CExp3Frame`：创建主窗口并设置标题；
- `CExp3App::InitInstance()`：
  - 初始化 MFC 运行时；
  - 创建 `Frame` 与 `View`；
  - 将 `View` 挂接为主窗口内容；
  - 显示并刷新窗口。

这部分保持最小化，只负责程序启动与窗口托管，不掺入图形算法逻辑。

### 11) 模块协作关系（总流程）

程序运行时完整链路如下：

1. `InitInstance` 创建窗口和视图；
2. `OnCreate` 启动定时器；
3. 每次 `OnTimer` 更新动画控制点；
4. `OnDraw` 用更新后的点集绘制控制线和 B 样条；
5. 周期循环，形成连续动画显示。

因此代码结构清晰对应实验要求：（1）B 样条图形显示 + （2）控制点二维变换动画。
